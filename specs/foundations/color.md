# Color

Цветовая система OES X. Два слоя: бренд-палитра (источник) и семантические
алиасы (то, что вы используете в компонентах). **В компонентах используйте
только семантические токены**, не бренд-цвета напрямую.

Все токены живут в [src/tokens.css](../../src/tokens.css). Светлая тема —
override-блок `[data-theme="light"]` там же.

## Surfaces

Подложки, отсортированы от самой глубокой (`--bg`) к выпуклой (`--surface-3`).

| Token         | Dark      | Light     | Применение                                  |
| ------------- | --------- | --------- | ------------------------------------------- |
| `--bg`        | `#06090F` | `#F0EEE9` | страница, корневой фон                      |
| `--bg-elev`   | `#0D1822` | `#FFFFFF` | приподнятые блоки (KPI-tile)                |
| `--surface`   | `#0F1620` | `#FFFFFF` | карточки, попоперы, модалки, чат-bubble bot |
| `--surface-2` | `#111A24` | `#F7F5F0` | hover state, скроллируемые секции, inputs   |
| `--surface-3` | `#131D28` | `#F5F4F0` | глубже surface-2 (active state, disabled bg)|
| `--nav-bg`    | rgba dark | rgba lite | шапка, с backdrop-filter blur               |

**Правило**: внутри карточки на `--surface` фон вложенных элементов —
`--surface-2`. Внутри `--surface-2` — `--surface-3`. Не нарушайте порядок.

## Foreground (текст и иконки)

| Token          | Dark      | Light     | Когда                                  |
| -------------- | --------- | --------- | -------------------------------------- |
| `--fg`         | `#E2EBF0` | `#0F1A14` | основной текст                         |
| `--fg-muted`   | `#7A98A8` | `#577070` | подписи, label, meta, secondary текст  |
| `--fg-dim`     | `#506680` | `#708888` | disabled, UI-линии, бледные иконки     |
| `--fg-on-teal` | `#050D0A` | `#FFFFFF` | текст поверх primary-фона (btn-primary)|

**WCAG-минимум**, который соблюдается:
- `--fg` на `--bg` / `--surface*` — AAA (>7:1) обе темы.
- `--fg-muted` на `--bg` — AA (≥4.58:1 в light, ≥5.58:1 в dark).
- `--fg-dim` — 3:1 для нетекстовых элементов (точки, линии). **НЕ для текста.**

## Kind chips (тип решения)

Для меток `Дашборд / Алерт / Команда`. Каждый — тройка `bg / fg / border`.

| Group        | bg / fg / b                                          |
| ------------ | ---------------------------------------------------- |
| `--chip-dash-*`  | синий — `Дашборд`                                |
| `--chip-alert-*` | оранжевый — `Алерт`                              |
| `--chip-app-*`   | teal — `Команда` / `Приложение`                  |

## Rules

- Никогда не используйте hex-литералы (`#1DB89A`, `rgba(29,184,154,…)`) в
  CSS-классах и в JSX `style={{...}}`. Только `var(--token-name)`.
- Исключение: внутри `tokens.css` — это и есть единственное место hex.
- Полупрозрачные оверлеи (тени модалок, backdrop) — допускается `rgba(0,0,0,…)`
  с явным комментарием, не нужно отдельный токен на каждый оттенок.
- При добавлении новой семантики (например, `--chip-foo-*`) — обновите этот
  файл, иначе следующая сессия не узнает.
