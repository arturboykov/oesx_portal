# Spacing

База — **4px**. Все отступы кратны 4. Это шкала «8-point grid с половинным
шагом», стандарт для плотного UI.

## Tokens

| Token   | Value | Когда                                                  |
| ------- | ----- | ------------------------------------------------------ |
| `--s-1` | 4px   | разделитель внутри inline-группы, gap иконки и текста  |
| `--s-2` | 8px   | gap кнопок, gap между чипами, padding чипа             |
| `--s-3` | 12px  | gap внутри карточки между связанными элементами        |
| `--s-4` | 16px  | gap в gridе KPI-tiles, отступ между независимыми блоками |
| `--s-5` | 20px  | внутренний padding карточки (по умолчанию)             |
| `--s-6` | 24px  | внешний padding страницы (по бокам)                    |
| `--s-8` | 32px  | вертикальный padding `.main` (сверху)                  |
| `--s-10`| 40px  | margin между крупными секциями                         |
| `--s-12`| 48px  | редко — большие воздухи в hero                         |
| `--s-16`| 64px  | bottom-padding `.main`                                 |

## Layout constants (НЕ часть spacing-шкалы, но живут в `tokens.css`)

| Token                    | Value | Назначение                          |
| ------------------------ | ----- | ----------------------------------- |
| `--sidebar-w`            | 240px | развернутая боковая панель          |
| `--sidebar-w-collapsed`  | 56px  | свёрнутая боковая панель            |
| `--nav-h`                | 52px  | верхняя шапка                       |
| `--grid-size`            | 48px  | размер ячейки фоновой сетки         |

## Inline padding/gap, не привязанные к шкале

Допустимо использовать px-значения inline, если они соответствуют шкале (5, 6, 7,
9, 10, 11, 14, 18, 22 — для тонких подгонок внутри компонентов). **Но**:

- Если число встречается в 3+ местах — заведите токен (или используйте `--s-*`).
- Если число ≥ 20 — почти наверняка должно быть из `--s-*`.

## Card-padding convention

| Компонент         | padding  |
| ----------------- | -------- |
| `.card`           | `20px`   |
| `.modal-head`     | `18px 22px` |
| `.modal-body`     | `22px`   |
| `.modal-foot`     | `14px 22px` |
| `.kpi-tile`       | `20px 22px` |
| `.popover`        | `6px` (содержит `.popover-item` со своим padding) |
| `.popover-item`   | `8px 12px` |
| `.btn`            | `7px 13px` |
| `.btn-sm`         | `5px 10px` |

## Page-level

```
.main         padding: 24px 32px 64px
.main-narrow  max-width: 1100px
.page-head    margin-bottom: 24px
.section-head margin: 32px 0 14px
```

## Rules

- **Не используйте `margin-*-auto` для центровки** — flex/grid с justify.
- **Не комбинируйте `margin` сверху и снизу одновременно** на соседних блоках —
  используйте только одно направление (предпочтительно `margin-top`) или `gap`
  у родительского flex/grid.
- **`gap` важнее `margin`** — если контейнер допускает flex/grid, используйте
  `gap`, а не отступы у потомков. Это упрощает добавление/удаление детей.
