# SolutionMeta

Метастрока решения — компактный набор технических метаданных под названием:
тип, версия, идентификатор, статус активности, автор, число запусков и (для
дашбордов) время последнего обновления данных.

Используется в двух местах:

- **Шапка просмотра решения** — [src/sections/solution-view.jsx](../../src/sections/solution-view.jsx) `SolutionHeader` (полная версия, все поля inline).
- **Строка таблицы «Решения»** — [src/sections/various.jsx](../../src/sections/various.jsx) `SolutionRow` (короткая версия, разнесённая по колонкам сетки).

## Purpose

Развести две сущности, которые ранее склеивались под лейблом «Обновлено»:

- **Время создания версии** = когда решение последний раз отредактировали.
  Источник: `OESDATA.versionMap[sol.id][i].date` (формат `'16.05.2026, 14:12'`).
- **Время обновления данных** = когда дашборд последний раз пересчитал данные.
  Источник: `sol.lastRun` (тот же формат).

Семантика лейблов:

| Лейбл | Что показывается | Где |
| --- | --- | --- |
| `v{N} · DD.MM.YYYY, HH:MM` | Номер версии + время её создания | Кнопка версии (шапка + таблица) |
| `Данные обновлены …` | `sol.lastRun` (или «ещё не обновлялись» для нового решения) | Только в шапке решений с `kind === 'dash'` |

Для `kind === 'alert'` и `kind === 'command'` в метастроке выводится только версия + дата её создания — `lastRun` не показываем.

## Anatomy (шапка просмотра)

```
[Тип-чип]  [v{N} · DD.MM.YYYY, HH:MM]  · id {shortId}  [· Данные обновлены {lastRun}]  · {runs} запусков  · автор {author}  · ● {активно|выключено}
            └── кнопка, открывает историю версий
```

Каждый сегмент (кроме первых двух) — `<span>` в один ряд, разделители «·» — часть текста, не отдельные ноды.

## Tokens used

- Шрифт: `var(--font-mono)` (вся метастрока, чтобы цифры/даты были выровнены).
- Размер: `font-size: 11px` для всех сегментов; `10px` — только для записи статуса в правом верхнем углу шапки (импорт из шапки).
- Цвет основного текста: `var(--fg-muted)`.
- Кнопка версии: фон `var(--teal-dim)`, граница `0.5px solid var(--border-strong)`, текст `var(--teal-400)`.
- Дата внутри кнопки версии — `opacity: 0.7`, чтобы визуально вторична по отношению к номеру.
- Точка активности: `var(--pos)` (включено) / `var(--fg-dim)` (выключено).

## Variants

| Variant | Триггер | Отличие |
| --- | --- | --- |
| `dash` | `sol.kind === 'dash'` | Дополнительный сегмент «· Данные обновлены …» в шапке |
| `alert` / `command` | `sol.kind !== 'dash'` | Сегмент «Данные обновлены» отсутствует |
| `historic` | смотрим v2 у решения с текущей v3 | Дата в кнопке версии — дата выбранной версии, не последней. Дополнительно справа от заголовка — оранжевый чип «ИСТОРИЧЕСКАЯ ВЕРСИЯ». |

## States

- **Свежий черновик** (`lastRun === '—'`, актуально для sol-07): сегмент «Данные обновлены ещё не обновлялись» (только для `dash`).
- **Без версий в `versionMap`**: кнопка деградирует до «v{N}» без даты (защита `versionDate &&`).
- **Не владелец**: имя автора без приписки «(вы)»; цвет имени `var(--fg)` вместо `var(--teal-400)`.

## Example

```jsx
// В SolutionHeader (solution-view.jsx)
const versions = OESDATA.versionMap[sol.id] || [];
const currentVersion = versions.find((x) => x.v === v) || versions[0];
const versionDate = currentVersion?.date;

<button onClick={onVersions} title="История версий" style={{...}}>
  <IconGitBranch size={10} /> v{v}
  {versionDate && <span style={{ opacity: 0.7, marginLeft: 4 }}>· {versionDate}</span>}
</button>

{sol.kind === 'dash' &&
  <span style={{ fontFamily: 'var(--font-mono)', fontSize: 11, color: 'var(--fg-muted)' }}>
    · Данные обновлены {sol.lastRun === '—' ? 'ещё не обновлялись' : sol.lastRun}
  </span>
}
```

## Notes

- Поле `sol.updated` (даты вида `'16.05.2026'`) больше не выводится в UI — `versionMap[…].date` точнее (с временем).
- Если когда-нибудь появятся свежие mock-данные с динамическим текущим временем — поверх можно добавить relative-time подпись («5 мин назад»). Сейчас даты статичны, относительный счётчик быстро устаревает и сбивает с толку.
